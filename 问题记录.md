## 本地图片准备指南

### 目录结构
```
backend/public/images/
├── cards/          # 卡片图片（精炼矿石时随机选取）
└── medals/         # 勋章图片（觉醒勋章时随机选取）
```

### 图片要求

| 项目 | 卡片 (cards/) | 勋章 (medals/) |
|------|---------------|----------------|
| **格式** | PNG / JPG / JPEG / WebP / SVG | PNG / JPG / JPEG / WebP / SVG |
| **推荐格式** | PNG（支持透明） | PNG（支持透明） |
| **尺寸** | 3:4 竖版（推荐 600×800px） | 1:1 正方形（推荐 800×800px） |
| **文件大小** | < 2MB | < 2MB |
| **命名** | 任意合法文件名，如 `card_01.png`、`milestone_dragon.webp` | 任意合法文件名，如 `medal_01.png`、`awakened_star.svg` |
| **数量** | 建议至少 5-10 张 | 建议至少 5-10 张 |
| **风格** | 魔法卡片、水晶、符文等 | 勋章、徽章、几何图案等 |

### 使用方式

**前端调用时传参控制：**
```javascript
// 使用本地图片（默认）
fetch('/api/cards/refine', {
  method: 'POST',
  body: JSON.stringify({
    walletAddress: address,
    oreIds: selectedOres,
    useAi: false  // ← 关键参数
  })
})

// 使用 AI 生成图片
fetch('/api/cards/refine', {
  method: 'POST',
  body: JSON.stringify({
    walletAddress: address,
    oreIds: selectedOres,
    useAi: true  // 或不传（默认 true）
  })
})
```

### 注意事项

1. **目录不能为空** — 如果目录中没有图片，系统会回退到 placehold.co 占位图
2. **文件名不含路径** — 返回的 URL 格式为 `/images/cards/xxx.png`，前端通过后端代理访问
3. **随机选取** — 每次精炼/觉醒时从目录中随机选一张，增加仪式感
4. **无需重启** — 添加/删除图片后即时生效（每次请求重新扫描目录）
