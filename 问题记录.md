## 本地图片准备指南

### 目录结构
```
backend/public/images/
├── cards/          # 卡片图片（精炼矿石时随机选取）
└── medals/         # 勋章图片（觉醒勋章时随机选取）
```

### 图片要求

| 项目 | 卡片 (cards/) | 勋章 (medals/) |
|------|---------------|----------------|
| **格式** | PNG / JPG / JPEG / WebP / SVG | PNG / JPG / JPEG / WebP / SVG |
| **推荐格式** | PNG（支持透明） | PNG（支持透明） |
| **尺寸** | 3:4 竖版（推荐 600×800px） | 1:1 正方形（推荐 800×800px） |
| **文件大小** | < 2MB | < 2MB |
| **命名** | 任意合法文件名，如 `card_01.png`、`milestone_dragon.webp` | 任意合法文件名，如 `medal_01.png`、`awakened_star.svg` |
| **数量** | 建议至少 5-10 张 | 建议至少 5-10 张 |
| **风格** | 魔法卡片、水晶、符文等 | 勋章、徽章、几何图案等 |

### 切换图片模式

在 `backend/.env` 中修改：
```env
USE_AI_IMAGES=false   # 使用本地图片
USE_AI_IMAGES=true    # 使用 AI 生成图片
```
修改后**重启后端**生效。

### 注意事项

1. **目录不能为空** — 如果目录中没有图片，系统会回退到 placehold.co 占位图
2. **文件名不含路径** — 返回的 URL 格式为 `/images/cards/xxx.png`，前端通过 Next.js 代理访问
3. **随机选取** — 每次精炼/觉醒时从目录中随机选一张，增加仪式感
4. **无需重启** — 添加/删除图片后即时生效（每次请求重新扫描目录）

## 后端环境变量速查

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `PORT` | 否 | `30001` | 后端服务端口 |
| `OPENAI_API_KEY` | **是** | 无 | AI API Key，配置后启用 AI 功能 |
| `OPENAI_API_BASE_URL` | 否 | `https://api.openai.com/v1` | API 基础地址，可替换为第三方兼容服务 |
| `CHAT_MODEL` | 否 | `gpt-4o-mini` | 聊天模型名称（矿石提炼/卡片标题/勋章标题） |
| `IMAGE_MODEL` | 否 | `dall-e-3` | 图像模型名称（卡片/勋章图片生成） |
| `USE_AI_IMAGES` | 否 | `false` | `true`=AI 生图，`false`=本地图片 |
| `AI_TIMEOUT` | 否 | `120000` | AI 请求超时时间（毫秒） |
| `SUPABASE_URL` | 否 | 无 | Supabase 项目地址 |
| `SUPABASE_ANON_KEY` | 否 | 无 | Supabase 匿名 Key |
| `PINATA_API_KEY` | 否 | 无 | Pinata IPFS Key |
| `PINATA_SECRET_KEY` | 否 | 无 | Pinata IPFS Secret |
