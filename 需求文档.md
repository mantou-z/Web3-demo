# Alcheme - 最终需求文档

> What you do and how you think build who you are. Define yourself, beyond the rules.

## 一、项目概述

### 1.1 核心理念
通过AI提取日常碎片的"灵光"，通过区块链技术将其凝练为不可篡改的SBT勋章。它不是一份简历，而是一面反映真实自我的"数字镜子"。

### 1.2 产品名称
**Alcheme** (炼金术)

### 1.3 核心价值
- 将日常碎石凝练成永久的勋章，提供正向反馈
- 成长过程可追溯、不可篡改
- 勋章可展示、可进化

## 二、产品设计

### 2.1 核心概念

```
用户输入 → AI提炼 → 矿石(Ore) → AI总结生图 → 卡片(Card) → AI总结生图 → SBT勋章(Medal)
```

### 2.2 灵光采集 (Mining)

**业务流程：** [欢迎页] → [矿山主页] → [浮窗输入] → [凿矿动画] → [AI结果列表] → [手动增/删/改] → [确认入库]

**核心功能：**
- 用户输入文字或语音
- AI实时提炼核心成就，生成"矿石"
- 用户可编辑、删除、添加AI提炼的结果
- 仪式化凿矿动画（最少3秒）
- 确认入库，矿石飞入仓库

**数据模型：**
```json
{
  "ores": [
    {
      "id": 1,
      "text": "成功在测试网部署首个合约",
      "color": "gold",
      "dimension": "Wisdom",
      "score": 5
    }
  ]
}
```

### 2.3 成长炼金 (Smelting)

**业务流程：** [进入精炼] → [炼金橱柜] → [勾选矿石] → [精炼动画] → [卡片显现] → [修改标题] → [收纳进库]

**核心功能：**
- 用户勾选多块矿石（无数量限制）
- AI总结共性并生图，生成个性化图片
- 卡片中央显示AI生成的图片
- 用户可修改卡片标题
- 消耗的矿石标记为"consumed"

**数据模型：**
```json
{
  "cardTitle": "合约炼金术士的初觉醒",
  "imageUrl": "ipfs://...",
  "consumedOreIds": [1, 2, 3, 4, 5]
}
```

### 2.4 身份铸造 (Forging)

**业务流程：** [进入觉醒] → [灵光阵] → [勾选卡片] → [觉醒动画] → [勋章显现] → [铸造/进化]

**核心功能：**
- **初次铸造：** 勾选N张卡片 → 点击"觉醒" → 铸造SBT勋章
- **追加进化：** 勾选1个已有勋章 + N张新卡片 → 点击"觉醒" → 进化勋章
- **错误校验：** 
  - 必须包含卡片
  - 如果包含勋章，则勋章只能选1个
  - 错误时法阵抖动，弹出提示

**数据模型：**
```json
{
  "title": "Web3探险家",
  "description": "历经30天Solidity学习，完成5个里程碑...",
  "imageUrl": "ipfs://...",
  "parentIds": [101, 102, 103]
}
```

### 2.5 Soul Profile（灵魂画像）

**模块A：灵魂雷达图**
- 用户自定义6个维度标签（如：韧性、代码、冥想）
- 每精炼一颗矿石，AI判定属于哪0-2个维度，该维度+1分
- 使用react-chartjs-2或recharts渲染雷达图

**模块B：动态勋章墙**
- 展示所有已铸造的3D勋章
- 点击勋章弹出详情卡
- 展示勋章背后的热力图和里程碑

## 三、技术架构

### 3.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js 14 + Tailwind CSS | 用户界面 |
| 动画 | Framer Motion | 交互动效 |
| Web3 | Wagmi + Viem | 链上交互 |
| 钱包 | RainbowKit + MetaMask | 钱包连接 |
| 后端 | Node.js + Express | API服务 |
| 数据库 | Supabase | 链下数据存储 |

| 合约 | Solidity (Foundry) | ERC-4906 Dynamic SBT |

### 3.3 后端API设计

**核心接口：**
```
POST   /api/analyze        # AI提炼矿石
POST   /api/refine         # AI总结生图（卡片）
POST   /api/awaken         # AI总结生图（勋章）
GET    /api/ores/:userId    # 获取用户矿石
GET    /api/cards/:userId   # 获取用户卡片
GET    /api/medals/:userId  # 获取用户勋章
GET    /api/medal/:id/trace # 追溯勋章背后的所有矿石
```

### 3.4 数据库设计

**Supabase表结构：**

```sql
-- 用户表
CREATE TABLE users (
  id UUID PRIMARY KEY,
  wallet_address TEXT UNIQUE,
  created_at TIMESTAMP
);

-- 矿石表
CREATE TABLE ores (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  raw_input TEXT,
  refined_data JSONB,
  status TEXT DEFAULT 'active', -- active, consumed
  created_at TIMESTAMP
);

-- 卡片表
CREATE TABLE cards (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  title TEXT,
  image_url TEXT,
  consumed_ore_ids UUID[],
  status TEXT DEFAULT 'active', -- active, consumed
  created_at TIMESTAMP
);

-- 勋章表
CREATE TABLE medals (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  title TEXT,
  description TEXT,
  image_url TEXT,
  token_id INTEGER,
  parent_ids UUID[], -- 来源卡片/勋章ID
  created_at TIMESTAMP
);

-- 用户灵魂画像表
CREATE TABLE user_profiles (
  user_id UUID PRIMARY KEY REFERENCES users(id),
  dimension1_name TEXT,
  dimension1_score INTEGER,
  dimension2_name TEXT,
  dimension2_score INTEGER,
  dimension3_name TEXT,
  dimension3_score INTEGER,
  dimension4_name TEXT,
  dimension4_score INTEGER,
  dimension5_name TEXT,
  dimension5_score INTEGER,
  dimension6_name TEXT,
  dimension6_score INTEGER
);
```

## 五、UI/UX设计

### 5.1 设计风格
- **主题色**：深色背景 + 魔幻渐变光效
- **风格**：神秘炼金术 + 游戏化体验
- **参考**：原神UI风格 + 塞尔达传说 + 梦幻童话感

**设计关键词：**
- 深邃的、带颗粒感的磨砂黑/紫
- 柔和的魔幻光晕
- 宝石般的渐变色彩
- 浮动的粒子效果
- 毛玻璃质感（Glassmorphism）

### 5.2 核心页面原型

**欢迎页**
- 纯黑底，弱光文字
- "What you do and how you think build who you are."
- 3秒自动淡出，进入矿山主页

**矿山主页**
- 中央：意念矿山（深色晶体堆砌，内部透出蓝紫色光芒）
- 右下角：炼金学徒剪影
- 引导语："你准备好开采今天的魔法矿石了吗？"
- 大按钮："开始开采"

**炼金实验室**
- 背景：深色石砖墙，中间铁质大锅，紫色药水冒泡
- 点击大锅：打开炼金橱柜（木质格架）
- 橱柜显示未使用的矿石列表

**灵光阵**
- 充满几何美感的发光法阵
- 点击法阵：打开炼金橱柜
- 橱柜两个页签：【里程碑卡片】和【已有的勋章】
