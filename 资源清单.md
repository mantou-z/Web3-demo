# Alcheme - 项目所需资源清单

## 一、AI API资源

### 1. OpenAI API
**用途：** 矿石提炼、卡片/勋章总结

**所需模型：**
- **GPT-4o-mini**：用于矿石提炼（成本低、速度快）
- **GPT-4o**：用于卡片/勋章逻辑总结（精度更高）
- **DALL-E 3**：用于生成卡片/勋章图片

**预估用量（48小时黑客松）：**
| 功能 | 调用次数 | 模型 | 预估成本 |
|------|----------|------|----------|
| 矿石提炼 | 100次 | GPT-4o-mini | $0.5-1 |
| 卡片总结 | 20次 | GPT-4o | $1-2 |
| 勋章总结 | 10次 | GPT-4o | $0.5-1 |
| 图片生成 | 30次 | DALL-E 3 | $3-5 |
| **总计** | | | **$5-9** |

**获取方式：**
- 注册OpenAI账户：https://platform.openai.com/
- 获取API Key
- 建议充值$10足够黑客松使用

**替代方案：**
- 使用Claude API（Anthropic）
- 使用国内大模型API（如通义千问、文心一言）

## 二、服务器资源

### 1. 后端服务器
**用途：** 运行Node.js API服务

**推荐方案：**
| 平台 | 免费额度 | 优点 | 缺点 |
|------|----------|------|------|
| **Vercel** | 100GB带宽/月 | 零配置部署、与Next.js完美集成 | 仅支持Serverless Functions |
| **Railway** | $5免费额度 | 支持长时间运行的服务 | 免费额度有限 |
| **Render** | 免费层 | 简单易用 | 冷启动慢 |
| **Fly.io** | 免费层 | 性能好 | 配置复杂 |

**推荐：** Vercel（前后端一起部署）

### 2. 数据库
**用途：** 存储链下数据（矿石、卡片、勋章）

**推荐方案：**
| 平台 | 免费额度 | 优点 | 缺点 |
|------|----------|------|------|
| **Supabase** | 500MB存储、50,000行 | PostgreSQL、自带API、实时订阅 | 免费项目7天不活跃会暂停 |
| **PlanetScale** | 5GB存储 | MySQL兼容、无服务器 | 免费层有限制 |
| **MongoDB Atlas** | 512MB存储 | NoSQL、灵活 | 需要自己写API |

**推荐：** Supabase（与规划文档一致）

## 三、存储资源

### 1. IPFS存储
**用途：** 存储卡片/勋章图片和元数据

**推荐方案：**
| 平台 | 免费额度 | 优点 | 缺点 |
|------|----------|------|------|
| **Pinata** | 1GB存储、100GB带宽 | 简单易用、API友好 | 免费额度有限 |

## 四、区块链资源

### 1. 测试网
**用途：** 部署智能合约、测试交易

**推荐网络：**
| 网络 | 优点 | 缺点 |
|------|------|------|
| **Avalanche Fuji** | 低Gas费、快速确认 | 需要AVAX测试币 |
| **Ethereum Sepolia** | 生态成熟 | Gas费较高 |
| **Base Sepolia** | Coinbase支持、低Gas | 相对较新 |

**推荐：** Avalanche Fuji（与规划文档一致）

**获取测试币：**
- Avalanche Fuji：https://faucet.avax.network/
- 需要少量AVAX测试币用于部署合约和测试

### 2. 钱包服务
**用途：** 社交登录、AA钱包

**推荐方案：**
| 服务 | 免费额度 | 优点 | 缺点 |
|------|----------|------|------|
| **Privy** | 免费层（5000 MAU） | 社交登录、AA钱包、简单集成 | 免费项目有品牌水印 |
| **Web3Auth** | 免费层 | 多链支持 | 配置复杂 |
| **Magic** | 免费层（1000 MAU） | 无密码登录 | 免费额度较少 |

**推荐：** Privy（与规划文档一致）

**获取方式：**
- 注册Privy账户：https://www.privy.io/
- 创建应用，获取App ID
