# Growth Forge - 成长锻造系统

> 让每一滴努力都成为可触摸的数字资产

## 一、项目概述

### 1.1 核心理念
通过Web3技术，将个人成长过程可视化、资产化。用户设定目标后，通过每日践行积累「矿石」，最终锻造出代表成就的「勋章」。

### 1.2 产品名称
**Growth Forge** (成长锻造)

### 1.3 核心价值
- 每一份坚持都有链上证明
- 成长过程可追溯、不可篡改
- 成就可展示、可交易

## 二、产品设计

### 2.1 核心概念

```
目标(Goal) → 每日践行 → 矿石(Ore) → 合成 → 卡片(Card) → 合成 → 勋章(Medal)
```

### 2.2 目标 (Goal)

用户创建的长期目标，是整个系统的起点。

| 属性 | 说明 |
|------|------|
| name | 目标名称（如：学会吉他） |
| description | 目标描述 |
| category | 分类（学习/健身/创作/社交） |
| oreType | 产出矿石类型 |
| createdAt | 创建时间 |
| status | 进行中/已完成/已放弃 |

**矿石类型定义：**
- 🔵 智慧矿石 (Wisdom) - 学习类目标
- 🔴 意志矿石 (Will) - 健身/习惯类目标
- 🟡 创造矿石 (Creation) - 创作类目标
- 🟢 连接矿石 (Connection) - 社交类目标

### 2.3 矿石 (Ore)

每次践行记录生成的NFT，是系统的基本单位。

| 属性 | 说明 |
|------|------|
| id | Token ID |
| goalId | 所属目标 |
| type | 矿石类型 |
| quality | 品质（1-5星，由记录内容质量决定） |
| content | 记录内容（链上存储hash） |
| timestamp | 记录时间 |
| streak | 当前连续天数 |

**品质机制：**
- ⭐ 一句话记录
- ⭐⭐ 简短描述
- ⭐⭐⭐ 详细记录
- ⭐⭐⭐⭐ 带感悟/反思
- ⭐⭐⭐⭐⭐ 深度复盘

### 2.4 卡片 (Card)

由同类型矿石合成，具有收藏和功能价值。

**合成规则：**
- 基础卡片：5颗同类型矿石
- 高级卡片：10颗同类型矿石 + 1张基础卡片
- 稀有卡片：30天连续打卡 + 20颗矿石

**卡片功能：**
| 卡片类型 | 功能 |
|----------|------|
| 基础卡片 | 展示收藏、解锁头像框 |
| 高级卡片 | 生成成长报告、铸造纪念NFT |
| 稀有卡片 | 解锁社区特权、获得治理权 |

### 2.5 勋章 (Medal)

代表目标达成的终极成就，由卡片合成。

**合成规则：**
- 铜勋章：3张基础卡片 + 目标完成30%
- 银勋章：2张高级卡片 + 目标完成60%
- 金勋章：1张稀有卡片 + 目标完成100%

**勋章价值：**
- 链上永久记录的成就证明
- 可在社交平台展示
- 未来可用于DAO治理权重

## 三、技术架构

### 3.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 智能合约 | Solidity + Foundry | 核心业务逻辑 |
| 区块链 | Ethereum Sepolia / Base | 测试网部署 |
| 前端 | Next.js + TailwindCSS | 用户界面 |
| Web3 | wagmi + viem | 链上交互 |
| 存储 | IPFS (Pinata) | 记录内容存储 |
| 钱包 | MetaMask | 用户连接 |

### 3.2 智能合约设计

```
contracts/
├── GrowthForge.sol      # 主合约
├── OreNFT.sol           # 矿石NFT
├── CardNFT.sol          # 卡片NFT
├── MedalNFT.sol         # 勋章NFT
└── ForgeEngine.sol      # 合成引擎
```

**GrowthForge.sol - 主合约**
```solidity
// 核心功能
- createGoal(name, description, category) => goalId
- recordDaily(goalId, contentHash) => oreTokenId
- synthesizeCards(oreIds[]) => cardTokenId
- forgeMedal(cardIds[], goalId) => medalTokenId
```

**OreNFT.sol - 矿石合约**
```solidity
struct Ore {
    uint256 goalId;
    OreType oreType;
    uint8 quality;
    string contentHash;
    uint256 timestamp;
    uint256 streak;
}
```

### 3.3 前端页面结构

```
pages/
├── index.tsx            # 首页 - 产品介绍
├── dashboard.tsx        # 仪表盘 - 我的目标
├── goal/[id].tsx        # 目标详情 - 记录践行
├── inventory.tsx        # 背包 - 矿石/卡片/勋章
├── forge.tsx            # 锻造台 - 合成页面
└── profile.tsx          # 个人主页 - 成就展示
```

### 3.4 数据流

```
用户操作 → 前端处理 → IPFS存储内容 → 合约调用 → 链上记录
```

**动画库：**
- 主动画库：[Framer Motion](https://www.framer.com/motion/) - React动画首选
- 粒子效果：[TSParticles](https://particles.js.org/) - 背景粒子
- Lottie动画：[LottieFiles](https://lottiefiles.com/) - 复杂交互动画

**必须实现的动效：**

| 场景 | 动效类型 | 实现方式 |
|------|----------|----------|
| 铸造矿石 | 粒子聚集+闪光 | Framer Motion + CSS |
| 合成卡片 | 卡牌翻转+光效 | CSS 3D Transform |
| 获得勋章 | 烟花+放大展示 | TSParticles + Framer Motion |
| 页面切换 | 渐变过渡 | Framer Motion AnimatePresence |
| 按钮悬停 | 微缩放+光晕 | CSS Transition |
| 加载状态 | 宝石旋转 | CSS Animation |

**Lottie动画需求：**
| 动画 | 时长 | 用途 |
|------|------|------|
| mining.json | 2s | 铸造矿石中 |
| forging.json | 3s | 合成卡片中 |
| achievement.json | 2s | 获得成就提示 |
| loading.json | loop | 页面加载 |
